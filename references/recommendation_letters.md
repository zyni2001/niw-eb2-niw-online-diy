# Recommendation Letters

## Use Two Different Roles

The strongest NIW set often includes:

1. a technically proximate recommender
2. an independent recommender

They should not sound like the same person with a different signature.

## Technically Proximate Recommender

This recommender can:

- describe the applicant's concrete contribution
- mention representative papers
- explain why the methods matter technically
- connect the work to real deployment needs

Good uses:

- Prong 2 support
- technical significance
- contribution-level validation

## Independent Recommender

This recommender should focus more on:

- broader engineering significance
- why the work matters beyond one lab or one employer
- why the field needs this kind of contribution
- why labor certification is a poor fit

Good uses:

- Prong 1 support
- Prong 3 balancing support
- independent validation that the work has broader value

## What makes a recommendation letter quotable

The best letters contain lines that can be directly inserted into the petition:

- "Research that makes AI behavior more secure, robust, and verifiable is essential to safe adoption."
- "These are not narrow academic concerns; they are engineering concerns."
- "His value is not tied to one narrow job description or one employer."

Encourage recommenders to include at least a few clean, quotable lines like that.

## Avoid

- generic praise with no factual basis
- repeating the CV without interpretation
- over-technical algorithm detail from independent recommenders
- making all letters read like the same template

## Technical-density rule

For USCIS, recommendation letters should usually emphasize:

- the problem
- the result
- the significance

more than:

- optimizer names
- benchmark names
- detailed algorithm internals

Keep numbers that show impact. Reduce jargon that requires domain knowledge.
